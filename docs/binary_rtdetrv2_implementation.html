What Changed

File Change Why
tools/prepare_binary_custom.py New dataset preparation script. It converts a custom YOLO-format dataset, reads labels, keeps only the selected foreground class, remaps it to category id 0, and writes COCO-style annotations. RT-DETRv2 already consumes COCO JSON through CocoDetection, so conversion keeps the project on its existing data path.
configs/dataset/custom_person_detection.yml New dataset config with num_classes: 1, remap_mscoco_category: False, and paths to person_train.json and person_val.json. Defines the binary detector task as person foreground versus background/no target.
configs/rtdetrv2/rtdetrv2_r18vd_binary_person_custom.yml New smoke-training model config based on RT-DETRv2 R18. It uses 160x160 images, batch size 2, one epoch, CPU-friendly settings, and COCO checkpoint tuning. Makes verification fast while still exercising the real detector, criterion, postprocessor, checkpoint loading, and evaluator.
references/deploy/rtdetrv2_torch.py Updated to add the repo root to sys.path, use eval_spatial_size from the config instead of hardcoded 640x640, and accept --threshold. Lets inference work with the small binary smoke config and local command-line execution.
.gitignore Ignores generated datasets, checkpoints, the local virtualenv, Python caches, and rendered result images. Keeps reproducible source files separate from heavy generated artifacts.

Procedure Diagram

Dataset Conversion

The custom dataset is generated at dataset/custom_binary. The converter produces:

  • annotations/person_train.json
  • annotations/person_val.json
  • images/train/ and images/val/ with converted custom images

Verification result: 2 train images, 2 validation images, and 2 person boxes from a synthetic custom YOLO dataset. Both splits contain one positive image and one negative/background image.

Commands Used

Prepare the binary dataset

python3 tools/prepare_binary_custom.py \
  --source-root /private/tmp/rtdetr_custom_raw \
  --target-class-id 0 \
  --target-class-name person \
  --force

Train on the custom dataset

python3 tools/train.py \
  -c configs/rtdetrv2/rtdetrv2_r18vd_binary_person_custom.yml \
  --device cpu \
  --seed 0

Run inference smoke test

python3 references/deploy/rtdetrv2_torch.py \
  -c configs/rtdetrv2/rtdetrv2_r18vd_binary_person_custom.yml \
  -r output/rtdetrv2_r18vd_binary_person_custom/best.pth \
  -f dataset/custom_binary/images/val/person_101.jpg \
  -d cpu \
  --threshold 0.01

The current training run is intentionally a small smoke test. It proves that the binary dataset, model head, checkpoint tuning, training loop, evaluator, and inference path work. It is not intended as a production-quality detector.

How to Adapt This Later

  • To target another class, rerun tools/prepare_binary_custom.py --source-root dataset/custom_raw --target-class-id 2 --target-class-name car --force and update the dataset config annotation filenames if the class name changes.
  • For better model quality, increase epoches, use a larger eval_spatial_size, and train on a larger custom dataset.
  • For GPU training, switch --device cpu to the available CUDA device and revisit batch size and AMP settings.
